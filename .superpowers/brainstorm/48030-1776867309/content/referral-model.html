Referral Mechanism — Which Model?

This shapes how complex the build is and how strong the growth loop is

A

Soft / Social

Personal share link + "Founding Ambassador" badge. Tracks who referred who. No billing changes. Build time: ~4 hours.

Pros

  • Ships before April 28
  • No Stripe complexity
  • Still measurable

Cons

  • Weak incentive to actually share
B

Hard / Economic

Referrer gets a free month for each converted paying subscriber. Stripe credit logic required. Build time: ~1 week.

Pros

  • Real incentive to share
  • Industry standard

Cons

  • Won't ship before launch
  • Billing edge cases
C

Founding-Exclusive Upgrade

Refer 3 paying users → unlock the next tier free for life. Only during founding window. Creates urgency to share before seats run out. Build time: ~2 days.

Pros

  • Strong "tell friends before it closes" hook
  • Exclusive to founding cohort — high perceived value
  • No ongoing billing credit complexity

Cons

  • Only works during founding window
  • Needs referral tracking in KV
